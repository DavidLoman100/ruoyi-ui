## 1. 获取OSS上传签名

| 字段名 | 类型 | 说明 |
|--------|------|------|
| accessId | String | AccessKey ID（访问密钥ID） |

| signature | String | 签名（用于验证请求合法性） |
| dir | String | 上传目录前缀（文件将保存到此目录） |
| host | String | OSS上传Host（上传目标地址） |

### 注意事项
- 签名有效期通常为1小时，过期后需要重新获取
- 文件路径结构：`{uploadDirPrefix}/{moduleType}/{YYYYMMDD}/{uuid}.{ext}`
- moduleType 需要与字典配置一致

## 2. 上传导入文件（服务端代理）

### 基本信息
- **接口路径**：`POST /file/fileImport/upload`
- **接口说明**：通过后端代理上传文件到OSS（适合小文件）
- **Operation ID**：`upload_1`
- **Tags**：通用文件导入
- **权限要求**：需要登录
- **Content-Type**：multipart/form-data

### 请求参数

#### Form Data 参数
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| file | File | 是 | 上传的文件对象 | trade.xlsx |
| bizType | String | 是 | 业务类型(import_biz_type)，如：stock、electricity | stock |

### 响应数据

#### 成功响应
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": 123456
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| data | Long | 导入记录ID（importId） |

### 注意事项
- 文件大小受服务器配置限制（通常10MB以内）
- 大文件建议使用前端直传OSS方式
- 上传成功后自动创建导入记录和文件详情

## 3. 创建导入记录（前端直传OSS后）

### 基本信息
- **接口路径**：`POST /file/fileImport/add`
- **接口说明**：前端直传OSS成功后，调用此接口创建导入记录和文件详情
- **Operation ID**：`createRecord`
- **Tags**：通用文件导入
- **权限要求**：需要登录
- **Content-Type**：application/json

### 请求参数

#### Request Body
**Schema**: `CreateImportRecordReq`

```json
{
  "bizType": "stock",
  "objectKey": "uploads/stock/20260413/abc123.xlsx",
  "fileName": "电量表7-8月部分.xlsx"
}
```

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| bizType | String | 是 | 业务类型(import_biz_type)，如：stock、electricity | stock |
| objectKey | String | 是 | OSS objectKey，前端直传OSS后的key | uploads/stock/20260413/abc123.xlsx |
| fileName | String | 是 | 用户选择的原始文件名 | 电量表7-8月部分.xlsx |

### 响应数据

#### 成功响应
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": 123456
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| data | Long | 导入记录ID（importId） |

#### 失败响应
```json
{
  "code": 500,
  "msg": "业务类型不能为空"
}
```

### 注意事项
- 前端需传入objectKey（OSS上的文件路径）和fileName（用户选择的原始文件名）
- 文件大小由后端从OSS自动获取，前端无需传递
- storedFileName（存储文件名）由后端从objectKey自动提取
- bucketName和ossHost由后端从配置自动获取
- 用户信息从SecurityContext自动获取
- 创建记录后系统会同时创建文件详情记录

## 6. 查询文件详情

### 基本信息
- **接口路径**：`POST /file/fileImport/detail/page/list`
- **接口说明**：查询导入记录关联的文件详情，查看每个文件的处理状态
- **Operation ID**：`queryDetails`
- **Tags**：通用文件导入
- **权限要求**：需要登录
- **Content-Type**：application/json

### 请求参数

#### Request Body
**Schema**: `QueryFileDetailReq`

```json
{
  "importId": 123456,
  "pageNum": 1,
  "pageSize": 10,
  "status": "success"
}
```

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| importId | Long | 是 | 导入记录ID | 123456 |
| pageNum | Integer | 否 | 页码，默认1 | 1 |
| pageSize | Integer | 否 | 每页大小，默认10 | 10 |
| status | String | 否 | 文件状态（import_file_status: success=成功, fail=失败） | success |

### 响应数据

#### 成功响应
**Schema**: `Page<FileDetailVo>`

```json
{

  "msg": "操作成功",
  "data": {
    "records": [
      {
        "detailId": 1001,
        "importId": 123456,
        "fileName": "电量表7-8月部分.xlsx",
        "storedFileName": "abc123.xlsx",
        "fileType": "xlsx",
        "fileSize": 102400,
        "bucketName": "dv-gulimall",
        "ossHost": "https://dv-gulimall.oss-cn-shenzhen.aliyuncs.com",
        "filePath": "uploads/stock/20260413/abc123.xlsx",
        "status": "success",
        "errorMsg": null,
        "createBy": "admin",
        "createTime": "2026-04-13T10:30:30",
        "updateBy": "admin",
        "updateTime": "2026-04-13T10:31:00",
        "remark": null
      }
    ],
    "total": 3,
    "size": 10,
    "current": 1,
    "pages": 1
  }
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| detailId | Long | 明细ID |
| importId | Long | 导入ID |
| fileName | String | 原始文件名（用户选择的文件名，如：电量表7-8月部分.xlsx） |
| storedFileName | String | 存储文件名（OSS上的UUID文件名，如：abc123.xlsx） |
| fileType | String | 文件类型(xlsx/xls/csv等) |
| fileSize | Long | 文件大小(字节) |
| bucketName | String | OSS存储桶名称 |
| ossHost | String | OSS访问域名 |
| filePath | String | OSS文件路径(含目录前缀) |
| status | String | 文件状态（import_file_status: success=成功, fail=失败） |
| errorMsg | String | 错误信息 |
| createBy | String | 创建者 |
| createTime | String | 创建时间 |
| updateBy | String | 更新者 |
| updateTime | String | 更新时间 |
| remark | String | 备注 |

### 注意事项
- 普通用户只能查看自己导入记录的文件详情
- 超级管理员可以查看所有文件详情
- 文件的完整URL可通过 `{ossHost}/{filePath}` 拼接获取（注意filePath已包含bucket前缀目录）
- 可通过 status 筛选文件
