## 1. 获取OSS上传签名

### 基本信息
- **接口路径**：`GET /fileImport/sign`
- **接口说明**：获取前端直传OSS所需的签名凭证
- **Operation ID**：`getSign_1`
- **Tags**：通用文件导入
- **权限要求**：需要登录
- **Content-Type**：application/x-www-form-urlencoded

### 请求参数

#### Query 参数
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| moduleType | String | 否 | 模块类型（字典配置），默认为default | stock |

### 响应数据

#### 成功响应
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "accessId": "LTAI5tL4g31NCmTQcrGoQuul",
    "policy": "eyJleHBpcmF0aW9uIjoiMjAyNi0wNC0xMlQxNzozMToxMS4wMDBaIiwiY29uZGl0aW9ucyI6W1siY29udGVudC1sZW5ndGgtcmFuZ2UiLDAsMTA3Mzc0MTgyNF0sWyJzdGFydHMtd2l0aCIsIiRrZXkiLCJ1cGxvYWRzL2Z1bmQvMjAyNjA0MTIvIl1dfQ==",
    "signature": "yP+yZq8xYDQDAIRCW5ts5+dsRRU=",
    "dir": "uploads/fund/20260412/",
    "host": "https://dv-gulimall.oss-cn-shenzhen.aliyuncs.com",
    "expire": 1775984731,
    "callbackUrl": null
  }

```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| accessId | String | AccessKey ID（访问密钥ID） |
| policy | String | 策略Base64编码（上传策略） |
| signature | String | 签名（用于验证请求合法性） |
| dir | String | 上传目录前缀（文件将保存到此目录） |
| host | String | OSS上传Host（上传目标地址） |
| expire | Long | 过期时间戳（秒级Unix时间戳） |
| callbackUrl | String | 回调地址（可选，当前未使用） |

### 注意事项
- 签名有效期通常为1小时，过期后需要重新获取
- 文件路径结构：`{moduleType}/{YYYYMMDD}/{uuid}.{ext}`
- moduleType 需要与字典配置一致

## 2. 上传导入文件（服务端代理）

### 基本信息
- **接口路径**：`POST /fileImport/upload`
- **接口说明**：通过后端代理上传文件到OSS（适合小文件）
- **Operation ID**：`upload_1`
- **Tags**：通用文件导入
- **权限要求**：需要登录
- **Content-Type**：multipart/form-data

### 请求参数

#### Form Data 参数
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| file | File | 是 | 上传的文件对象 | trade.xlsx |
| dataType | String | 是 | 数据类型/模块类型，如：stock、fund | stock |

### 响应数据

#### 成功响应
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": 123456
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| data | Long | 导入记录ID（importId） |

### 注意事项
- 文件大小受服务器配置限制（通常10MB以内）
- 大文件建议使用前端直传OSS方式
- 上传成功后自动创建导入记录并异步处理

## 3. 创建导入记录（前端直传OSS后）

### 基本信息
- **接口路径**：`POST /fileImport/record`
- **接口说明**：前端直传OSS成功后，调用此接口创建导入记录
- **Operation ID**：`createRecord`
- **Tags**：通用文件导入
- **权限要求**：需要登录
- **Content-Type**：application/json

### 请求参数

#### Request Body
**Schema**: `CreateImportRecordReq`

```json
{
  "fileUrl": "https://your-bucket.oss-cn-hangzhou.aliyuncs.com/stock/20260411/abc123.xlsx",
  "dataType": "stock"
}
```

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| fileUrl | String | 是 | OSS文件完整URL | https://... |
| dataType | String | 是 | 数据类型/模块类型，如：stock、fund | stock |

### 响应数据

#### 成功响应
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": 123456
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| data | Long | 导入记录ID（importId） |

#### 失败响应
```json
{
  "code": 500,
  "msg": "文件URL不能为空"
}
```

### 注意事项
- 文件名和大小会自动从OSS获取，无需前端传递
- 用户信息从SecurityContext自动获取
- 创建记录后系统会异步处理文件解析

### 基本信息
- **接口路径**：`POST /fileImport/records/query`
- **接口说明**：分页查询导入历史记录
- **Operation ID**：`queryRecords`
- **Tags**：通用文件导入
- **权限要求**：需要登录
- **Content-Type**：application/json

**Schema**: `QueryImportRecordReq`

```json
{
  "pageNum": 1,
  "pageSize": 10,
  "dataType": "stock",
  "status": "2",
  "userId": 1001
}
```

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| pageNum | Integer | 否 | 页码，默认1 | 1 |
| pageSize | Integer | 否 | 每页大小，默认10 | 10 |

| status | String | 否 | 状态 | 2 |
| userId | Long | 否 | 用户ID（超管可指定查询某用户） | 1001 |

```json
        "successRate": "95%"

    ],
    "total": 50,
    "size": 10,
    "current": 1,
    "pages": 5
  }
}
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| importId | Long (int64) | 导入ID |
| fileId | Long (int64) | 文件ID |
| userId | Long (int64) | 用户ID |
| assetType | String | 数据类型 |
| totalRows | Integer (int32) | 总行数 |
| successRows | Integer (int32) | 成功行数 |
| failRows | Integer (int32) | 失败行数 |
| status | String | 状态 |
| errorMsg | String | 错误信息 |
| createTime | String (date-time) | 创建时间 |
| updateTime | String (date-time) | 更新时间 |
| successRate | String | 成功率 |

### 注意事项
- 普通用户只能查询自己的导入记录
- 超级管理员可以查询所有用户或指定用户的记录
- 状态筛选：0-待处理、1-处理中、2-成功、3-失败、4-部分成功

## 附录：状态码说明

### 导入记录状态（status）
| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待处理 | 文件已上传，等待处理 |
| 1 | 处理中 | 正在解析文件 |
| 2 | 导入成功 | 所有数据导入成功 |
| 3 | 导入失败 | 所有数据导入失败 |
| 4 | 部分成功 | 部分数据成功，部分失败 |

### 导入明细业务状态（businessStatus）
| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待处理 | 等待处理 |
| 1 | 处理中 | 正在处理 |
| 2 | 成功 | 业务处理成功 |
| 3 | 失败 | 业务处理失败 |
