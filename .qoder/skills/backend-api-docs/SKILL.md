# 后端文档查询（接口文档 + 业务流程）

## ⚠️ 重要说明

**`doc/api` 目录下的文档是后端提供的接口文档，前端只读不写！**

- ✅ **可以做的**：读取文档、理解接口规范、根据文档实现前端代码
- ❌ **禁止做的**：修改文档内容、调整接口定义、变更参数说明
- 📝 **文档更新**：由后端开发人员维护，前端发现文档问题应反馈给后端

## 功能说明

此 skill 用于从项目的 `doc/api` 目录中搜索和获取后端文档，包括：
- **API 接口文档**：接口定义、参数说明、返回值格式
- **业务流程文档**：业务逻辑、处理流程、实现细节

## 使用场景

- 查找特定 API 接口的定义和参数
- 了解业务功能和实现流程
- 查看接口请求/响应格式
- 搜索特定模块的业务逻辑说明
- 理解功能的完整业务流程

## 搜索流程

### 1. 定位文档目录

所有后端 API 文档位于：
```
doc/api/
```

### 2. 搜索文档

使用以下方法搜索文档：

**精确搜索**：
- 使用 `search_file` 工具搜索特定文件名的 `.md` 文件
- 示例：`文件导入-业务流程.md`、`文件导入-接口文档.md`

**关键词搜索**：
- 使用 `grep_code` 工具在 `doc/api` 目录下搜索关键词
- 示例：搜索包含"上传"、"导入"、"用户"等关键词的文档

**目录浏览**：
- 使用 `list_dir` 工具查看 `doc/api` 目录结构
- 了解有哪些模块的文档可用

### 3. 读取文档内容

使用 `read_file` 工具读取找到的文档文件，获取完整内容。

## 常用命令示例

### 查看所有 API 文档
```
list_dir(path: "doc/api")
```

### 搜索包含特定关键词的文档
```
grep_code(path: "doc/api", regex: "关键词", glob: "*.md")
```

### 读取特定文档
```
read_file(file_path: "doc/api/模块名/文档名.md")
```

## 文档组织结构

文档按模块组织在 `doc/api` 下的子目录中，例如：
- `doc/api/dv-file/` - 文件相关模块
- 其他模块目录...

每个模块目录下可能包含：
- `XXX-业务流程.md` - 业务流程说明（业务逻辑、处理步骤、实现细节）
- `XXX-接口文档.md` - 接口定义文档（API 参数、返回值、错误码）
- 其他相关文档

## 响应格式

找到文档后，应提供：
1. **文档路径**：完整的文件路径
2. **文档类型**：接口文档 或 业务流程文档
3. **文档标题**：从文档内容提取的标题
4. **内容摘要**：简要说明文档的主要内容
5. **关键信息**：提取用户关心的核心内容

**对于接口文档**，重点关注：
- 接口路径和请求方法
- 请求参数和格式
- 返回值结构
- 错误码说明

**对于业务流程文档**，重点关注：
- 业务场景和使用场景
- 处理流程和步骤
- 关键业务逻辑
- 注意事项和边界条件

## 注意事项

- 🚨 **只读不写**：`doc/api` 目录下的文档是后端提供的，前端禁止修改
- 只搜索 `doc/api` 目录下的 `.md` 文件
- 如果未找到相关文档，列出可用的文档供用户选择
- 对于大型文档，先提供摘要，用户需要时再提供完整内容
- 发现文档问题或与实现不符，应反馈给后端开发人员
